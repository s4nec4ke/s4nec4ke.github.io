# site/ — личный сайт-портфолио Андрея

## Что это
Одностраничник + страница CV. Цель — отправлять ссылку HR **вместе с резюме**, чтобы создать вау-эффект и показать продуктовое мышление через подачу.

## Главная страница
- `index.html` — основной файл.
- **Дизайн-база — `prototypes/hybrid-dark.html`** (тёмная тема, палитра mint/cyan/magenta, типографика Geologica + JetBrains Mono + Caveat). Это эталон визуала, а не жёсткое требование по структуре файлов.
- Стили можно держать инлайном или вынести в `styles.css` / `script.js` — выбор по удобству задачи.

## cv/index.html — снапшот-референс
- Та же база hybrid-dark, **уже расширенная** реализацией раскрывающихся карточек (`case-trigger` / `case-panel`, slide-in).
- Держим как точку отсчёта: с чем сравнивать итоговый `index.html`, откуда подсмотреть готовую механику раскрытия для секции «Мои проекты».
- Сам файл не трогаем без явной просьбы.

## Контент по проектам
- `projects/{slug}/context.md` — источник правды для секций «Мои проекты» на главной. Слаги: `magic-pot`, `oqtacore`, `ai-jobsearch`.
- Скриншоты/демо проектов → `projects/{slug}/assets/`.
- При сборке раскрывающихся карточек проектов **читать context.md этого проекта**, а не выдумывать тексты.
- Паттерн повторяет `vacancies/items/{id}/` из главного проекта — тонкий индекс + длинные тексты в подпапке.

### Ассеты: reference vs production
Внутри `projects/{slug}/assets/` могут быть два разных типа изображений — **проверять пометку в `context.md` каждого проекта** перед использованием на сайте:
- **production-ready** — авторские арты/изображения самого проекта, можно подключать прямо в `index.html` как готовые картинки (пример: `magic-pot/assets/` — гномы, постеры, стикеры).
- **reference-only** — скриншоты чужих сайтов / референсы для контекста. **Не вставлять** в `index.html`, использовать только для понимания дизайна и тона (пример: `oqtacore/assets/` — скрины с oqtacore.com).

## Ассеты
- `assets/arts/` — авторские арты (аватарка, постеры типа `partners-in-crime.png`, `vault-13.png` и т.п.).
- `assets/stickers/` — мелкие декоративные стикеры.
- Перед тем как просить новые картинки — проверить, нет ли подходящих здесь.

## Прототипы
- `prototypes/` — референсные макеты разных направлений (classic, asymmetric, typographic, memecoin и т.п.). **Не трогать без явной просьбы**, не использовать как источник для index.html кроме `hybrid-dark.html`.

## Язык
- Сайт на английском (целевая аудитория — иностранные HR/founders). Комментарии в коде и тексты в `projects/*/context.md` — можно на русском.
