## Что это и для кого
Magic Pot — мини-приложение в Telegram для создания **децентрализованных призовых пулов**. Любой юзер, проект или комьюнити может собрать кастомизируемый пул на любых токенах TON: один или несколько победителей, burn-механика, реферальные награды, условия участия (подписка на канал и т.п.). Распределение наград — автоматически через смарт-контракты.

**Для кого:**
- **Web3-проекты** — инструмент engagement и активации комьюнити без затрат на «классические» маркетинговые активности.
- **Telegram-комьюнити** — способ привлечь живую аудиторию через гивэвеи с фильтрацией ботов (привязка к TON-кошельку + Telegram-аккаунту).
- **Холдеры токенов** — новый utility для токенов, лежащих мёртвым грузом после листинга.

## Стек / технологии
- **Блокчейн:** TON (The Open Network)
- **Платформа:** Telegram Mini App
- **Auth:** Ton Connect (универсальный open-standard для TON-кошельков) — UX «3 тапа из чата комьюнити»
- **Smart-contracts:** для пулов, распределения, burn-механик. Правила вшиты в код, не меняются после старта игры
- **Архитектура:** **non-custodial** — фонды не хранятся на сервисных кошельках, всё проходит через смарт-контракт. Пользователь подписывает транзакцию, контракт сам списывает токены
- **Бот в чатах:** официальный Magic Pot bot можно добавить в комьюнити-чат → real-time апдейты по пулу (рост призового фонда, сброс таймера, кто сходил последним)
- **Прозрачность:** все транзакции — on-chain, верифицируемы через Tonscan

## Продуктовые механики (3 формата игр)

Три разных сценария пулов — каждый закрывает свою задачу комьюнити.

### 🎲 Pot — классический «горшок»
Игроки «инвестируют» небольшую сумму выбранного токена в общий пул, соревнуясь за главный приз.
- **Таймер:** от 5 минут до 24 часов, **сбрасывается на максимум** при каждой новой транзакции (механика «last-mover wins»)
- **Размер взноса:** фиксированный или **растущий** на N% с каждой транзакцией
- **Призы:** 1–99 победителей, случайно или из последних участников + опциональные реферальные награды + опциональный **burn N%** для дефляционного эффекта

### ⚔️ Clash — битва комьюнити
Формат «команда на команду»: два (или больше) комьюнити соревнуются «дойти до сокровища» первыми, накапливая транзакции.
- Несколько организаторов сбрасывают токены в общий пул
- Цена каждой следующей попытки одного игрока **растёт** (анти-фарм баланс)
- Победитель забирает всё — включая токены проигравших
- Опции: токены проигравшей стороны можно **сжечь** или **редистрибьютить** победителям

### 🔥 Firebox — игра «против организатора»
Организатор кладёт приз в **стейблкоине** (USDT), игроки тратят свои community-токены на попытки «найти сокровище».
- Организатор задаёт сумму USDT, цену попытки и вероятность находки
- Все потраченные на попытки токены **сжигаются** по правилам игры
- Опция: приз можно делить на части (несколько победителей)
- Опция: цена попытки растёт с каждой транзакцией одного игрока

> **Зачем три формата:** каждый закрывает свой engagement-сценарий — Pot для retention внутри одного комьюнити, Clash для коллабораций между проектами, Firebox для проектов с большой казной в стейблах и потребностью в burn'е своего токена.

## Челленджи и как решал

### 1. Боты в гивэвеях (типичная проблема Web2-крипто)
**Проблема:** стандартные TG-гивэвеи привлекают ботов, метрики «надуваются», реальный engagement нулевой.
**Решение:** связка «Web3-кошелёк + TG-аккаунт» — для участия нужен реальный кошелёк с балансом. Весь процесс трекается on-chain через Tonscan.

### 2. Token utility после листинга
**Проблема:** токены проектов лежат в кошельках без применения после listing-хайпа.
**Решение:** Magic Pot даёт **любому холдеру** инструмент использовать токены — создавать пулы, проводить гивэвеи, burn-механики. Utility идёт не от проекта, а от комьюнити.

### 3. User acquisition без бюджета
**Решение:** четыре параллельных канала:
- коллаборации с Web3-проектами (мы покупаем их токены и сами спонсируем пулы — обе стороны видят value)
- коллаборации с TG-комьюнити без токенов (спонсорство гивэвеев в USDT)
- собственное комьюнити с popular memecoins
- airdrop early users → points за активность до листинга токена

### 4. Найм в ранней экосистеме TON
**Проблема:** на старте экосистемы TON опытных людей мало, два специалиста из команды выпали в один момент.
**Решение:** закрыл **2 замены за 1 неделю** через нетворк в TON-комьюнити, спринт не сдвинулся.

### 5. Приоритизация под TON Accelerator
**Ситуация:** к питчу/демо в акселераторе нужен был работающий продукт с понятной механикой, не разнокалиберный набор фич.
**Решение:** срезал скоуп игровых сценариев до одного формата (Pot), остальные форматы (Clash, Firebox) добавлял итеративно после релиза.
**Результат:** запустились в срок, **получили грант TON Accelerator**.
